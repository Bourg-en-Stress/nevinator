Les licences open source c'est la jungle !

Bienvenue



Quand vous créer un repository github vous pouvez choisir d'attribuer une licence ou de choisir "none" (= sans licence).
Oui, mais quand on y regarde de plus prés c'est une vraie jungle et il n'est pas toujours évident de s'y retrouver, de faire un choix. Avec ce site, vous êtes au bon endroit !
Nous allons, au travers d'un questionnaire, vous aider à trouver une licence adaptée.

Vous voulez choisir "None" ? C'est votre choix, mais attention lisez bien ce qui suit :

 lorsque vous réalisez une œuvre créative (ce qui inclut le code), cette œuvre est soumise par défaut à un droit d'auteur exclusif.
À moins que vous choisissiez une licence spécifiant le contraire, personne d'autre ne peut copier, distribuer ou modifier votre travail sans que vous soyez victime d'un vol, d'un détournement ou d'un litige.
Interdire l'utilisation de votre code n'est peut-être pas ce que vous entendez par "none". Une licence open source permet la réutilisation de votre code tout en conservant les droits d'auteur. Si votre objectif est de vous affranchir complètement des restrictions de droits d'auteur, essayez plutôt une licence Créative Commons.
